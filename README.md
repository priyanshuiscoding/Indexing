# ⚖️ Court File Indexer v2 — RAG System

## How it works
1. You upload a PDF → backend extracts text from every page (digital or OCR)
2. Text is stored in a local vector database (ChromaDB)
3. Index is generated by asking the AI to reason over the extracted text
4. You can also chat with the document in Hindi or English

---

## WHAT YOU NEED TO INSTALL FIRST

### 1. Python 3.10 or higher
Download from: https://www.python.org/downloads/
- On Windows: check "Add Python to PATH" during install
- Verify: open a terminal and type `python --version`

### 2. Node.js 16 or higher
Download from: https://nodejs.org (choose LTS)
- Verify: open a terminal and type `node --version`

### 3. Tesseract OCR (for scanned PDFs)
This is needed to read text from scanned/image-based PDFs.

**Windows:**
- Download installer: https://github.com/UB-Mannheim/tesseract/wiki
- Run the installer → on the "Choose Components" screen, expand "Additional language data" and check "Hindi"
- Install to the default path: `C:\Program Files\Tesseract-OCR\`
- After installing, add to PATH:
  - Search "Environment Variables" in Windows search
  - Edit "Path" under System variables
  - Add: `C:\Program Files\Tesseract-OCR\`

**Mac:**
```
brew install tesseract
brew install tesseract-lang
```

**Linux (Ubuntu):**
```
sudo apt install tesseract-ocr tesseract-ocr-hin tesseract-ocr-eng
```

Verify: `tesseract --version`

## SETUP STEPS

### Step 1 - Create your backend `.env` file

Go into `court-rag/backend/` and create a file named `.env`:

```
LOCAL_LLM_BASE_URL=http://127.0.0.1:11434
LOCAL_TEXT_MODEL=qwen2.5:14b
LOCAL_VISION_MODEL=qwen2.5vl:7b
LOCAL_LLM_TIMEOUT=180
CHROMA_DB_PATH=./chroma_db
PDF_STORAGE_PATH=./stored_pdfs
INDEX_EXPORT_PATH=./index_exports
TESSERACT_LANG=hin+eng
ENABLE_HANDWRITTEN_HINDI_ASSIST=true
```

### Step 2 — Install Python dependencies

Open a terminal in the `court-rag/backend/` folder:

```bash
pip install -r requirements.txt
```

This will take 5–10 minutes the first time. It downloads:
- FastAPI (web server)
- PyMuPDF (PDF reading)
- Pytesseract (OCR)
- ChromaDB (vector database)
- sentence-transformers (local embedding model — ~90MB download)

### Step 3 — Install frontend dependencies

Open a terminal in the `court-rag/frontend/` folder:

```bash
npm install
```

## RUNNING THE APP

You need TWO terminal windows open at the same time.

### Terminal 1 — Start the backend

```bash
cd court-rag/backend
python -m uvicorn main:app --reload --port 8000
```

You should see:
```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Loading embedding model...
INFO:     Embedding model ready
INFO:     Server ready
```

### Terminal 2 — Start the frontend

```bash
cd court-rag/frontend
npm start
```

The browser will open automatically at: **http://localhost:3000**

## USING THE APP

### Upload a PDF
- Click "Upload PDF" button
- The backend will:
  - Extract text from every page (digital PDFs: instant, scanned PDFs: OCR takes ~1-2s per page)
  - Store all text in the local vector database
  - Generate the index automatically
- You will see the index table fill in on the right

### Index Table
- Every document/section found is listed with source badge:
  - **TOC** = found in the table of contents page
  - **AI** = detected automatically by scanning page text
  - **GAP** = placeholder for uncovered pages
  - **MAN** = manually added by you
- Click any row → jumps to that page in the PDF viewer
- Click ✎ to edit an entry inline
- Click 🗑 to delete an entry
- Click "+ Add Entry" to add a new one manually
- Click "↺ Re-index" to regenerate the index if you want

### Document Chat
- Click the "Document Chat" tab
- Ask anything about the document in Hindi or English
- Examples:
  - "What documents are on pages 30 to 60?"
  - "पृष्ठ 15 पर क्या है?"
  - "List all affidavits in this file"
  - "What is the case number?"
- Blue page number buttons appear in the answer → click to jump to that page

## TROUBLESHOOTING

### "tesseract is not installed or not in PATH"
- Windows: make sure you added `C:\Program Files\Tesseract-OCR\` to PATH and restarted the terminal
- Run `tesseract --version` to check

### "No module named 'fitz'"
```bash
pip install pymupdf
```

### Backend starts but shows embedding model error
The sentence-transformers model downloads automatically on first run (~90MB).
If it fails, check your internet connection and run again.

### "Local text model request failed"
Make sure Ollama is running and that the model names in `backend/.env` are already pulled locally.

### "Local vision model request failed"
The vision model is missing or not loaded. Pull the model from the Ollama terminal and restart the backend.

### Frontend can't reach backend (network error)
Make sure the backend is running on port 8000. Check Terminal 1.

### Tesseract Hindi language not found
- Windows: re-run the Tesseract installer and check "Hindi" under Additional Language Data
- Linux: `sudo apt install tesseract-ocr-hin`
- Mac: `brew install tesseract-lang`

### Port already in use
```bash
# Kill port 8000 (backend)
npx kill-port 8000

# Kill port 3000 (frontend)  
npx kill-port 3000
```
